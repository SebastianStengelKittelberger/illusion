Bereich 01 — UKey Explorer

Den Überblick behalten:
Was ist zugeordnet, was fehlt?

Jedes Produktdatenfeld hat einen UKey (eindeutigen Bezeichner). Der Explorer zeigt auf einen Blick, welche Felder der Datenbank bereits einer Webseiten-Darstellung zugeordnet sind — und welche noch nicht.

  • ✓ Grüne Felder → bereits zugeordnet und in Verwendung
  • ✓ Rote Felder → vorhanden in Datenbank, aber noch nicht auf der Seite
  • ✓ Filter nach einzelner SKU oder gesamtem Sortiment
  • ✓ Direktlink zum Mapping-Editor für jedes Feld

Bereich 02 — Mapping-Regeln

Die Zuordnungsregeln:
Datenbankfeld → Webseitenfeld

Die Mapping-Config legt fest, welches Feld aus der Produktdatenbank wo und wie auf der Webseite erscheint. Einmal angelegt, gilt die Regel für alle Produkte automatisch.

  • ✓ Übersichtliche Tabelle aller Zuordnungsregeln
  • ✓ Regeln anlegen, bearbeiten, kopieren, löschen
  • ✓ Import / Export als JSON-Datei
  • ✓ „Alle anwenden" startet eine Neu-Indexierung
  • ✓ Unterstützt einfache Texte, Bilder, Varianten, komplexe Berechnungen

Bereich 03 — Template Editor

Seitenaufbau und Vorlagen
zentral verwalten.

Der Template Editor verwaltet den Aufbau der Produktseiten. Seiten bestehen aus Vorlagen (z.B. Produktbild, Beschreibung, Technische Daten), die flexibel kombiniert werden können.

  • ✓ Alle Seiten und Vorlagen in der linken Übersicht
  • ✓ Vorlagen per Drag & Drop auf Seiten zuweisen
  • ✓ Versionshistorie: Jede Änderung gespeichert, Wiederherstellung per Klick
  • ✓ Labels (Übersetzungen) zentral verwalten
  • ✓ Live-Vorschau parallel zum Editor
  • ✓ Undo / Redo

Beispiel: Die Seite „stage-produktseite" besteht aus 5 Vorlagen — Header, Bild, Beschreibung, Technische Daten, Zubehör. Jede Vorlage kann unabhängig bearbeitet werden ohne die anderen zu beeinflussen.

Bereich 04 — Visual Edit Mode ★

Klick auf die Seite.
Wert direkt ändern.

Im Visual Edit Mode sieht der Nutzer die fertige, befüllte Produktseite. Jedes Datenfeld ist mit einem gestrichelten Rahmen markiert. Ein Klick öffnet einen Dialog zur direkten Änderung der Zuordnung.

1️⃣

Auf einen Wert klicken

z.B. auf „220V" auf der Produktseite

2️⃣

Neues Feld auswählen

Aus der Liste aller verfügbaren Datenbankfelder wählen

3️⃣

Sofort gespeichert

Die Vorlage wird aktualisiert, Vorschau lädt neu. Kein Entwickler nötig.

Bereich 05 — Datenqualitäts-Dashboard

Probleme sehen bevor
der Kunde sie meldet.

Das Dashboard zeigt für jedes Datenfeld, wie vollständig es befüllt ist — über alle Produkte hinweg. Fehlende Werte, defekte Bilder oder leere Beschreibungen werden sofort sichtbar.

  • ✓ Vollständigkeits-Score pro Feld (0–100 %)
  • ✓ Liste aller Produkte mit fehlendem Wert
  • ✓ Direktlink zum Mapping-Editor für schnelle Korrektur
  • ✓ Kritische Felder (unter 50 %) rot markiert
  • ✓ Filterbar nach Feldname

Beispiel: Das Feld „Produktbild" ist bei 8 von 10 Produkten befüllt — die 2 fehlenden werden per Klick angezeigt. Kein manuelles Durchklicken aller Produktseiten mehr.

Bereich 06 — URL-Routing

Welche Adresse zeigt
welche Seite — pflegbar.

Die URL-Routing-Tabelle legt fest, welche Web-Adresse welche Seite anzeigt. Neue Seiten können ohne Entwickler-Eingriff live gestellt werden.

  • ✓ Beliebige URL-Pfade anlegen und zuordnen
  • ✓ Produktseiten: SKU automatisch aus der URL lesen
  • ✓ CMS-Seiten: reine Textseiten ohne Produktdaten (z.B. Über uns, Kontakt)
  • ✓ Direktes Testen der URL per Klick
  • ✓ Änderungen sofort wirksam — kein Deployment
